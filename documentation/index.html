What Does It Do?

This library is an add-on for SuperScript. It is strongly recommended to read the documentation for the core project before proceeding with this library. If you haven't yet read the core documentation then it is recommended that you start by reading about the core concepts.

SuperScript.JavaScript.Mvc offers a Razor view-based block for enclosing server-side entities to be processed by SuperScript.

SuperScript will retrieve the contents of this block and create an instance of CollectedScript containing these contents.

When the Razor view is rendered as a webpage the contents of the block will be empty and the retrieved contents will be emitted through the referenced IEmitter. 

	            public static class HtmlExtensions
	            {
	                static IDisposable JavaScriptContainer(this HtmlHelper helper)
	                static IDisposable JavaScriptContainer(this HtmlHelper helper, string emitterKey)
	                static IDisposable JavaScriptContainer(this HtmlHelper helper, int insertAt)
	                static IDisposable JavaScriptContainer(this HtmlHelper helper, string emitterKey, int insertAt)
	            }
Name Type Description
helper HtmlHelper

The context System.Web.Mvc.HtmlHelper object exposed on a Razor view.

emitterKey string

Indicates which instance of IEmitter the content should be added to.

Optional. If not specified then the contents will be emitted through the default instance of IEmitter.

insertAt int

Indicates the index in the collection at which the contents are to be inserted.

Optional. If not specified, like all declarations, this declaration will be appended to the Collection.

Note that a declarable may be added at the specified index in the Collection at the time when this declaration is being processed; this is notwithstanding any later declarations which may also be inserted at the same index.

Example

			@using SuperScript.JavaScript.Mvc.Containers
			…
			@using (Html.JavaScriptContainer("javascript", 0))
			{
				<script type="text/javascript">
				
					var demoVar = "hello world";
				
				</script>
			}

The above example will move the inner <script> block into a CollectedScript declaration. In this example the declaration will be emitted through the an IEmitter with the Key "javascript". (Note that this requires that an IEmitter with the Key "javascript" is configured within the project. This is not provided out of the box by SuperScript.)

The declaration created in the above example will be added as the first item in the Collection. Note that this declarable will only be assured as the first item in the Collection at the time it is being processed; this is notwithstanding any later declarations which may also be inserted at the same index.

The Contents

The contents of the block are not required to be JavaScript. SuperScript will simply move the contents into a CollectedScript declaration. It is left to the developer to configure how and where these contents are emitted.

In the above examples JavaScript was clearly used. It is not necessary to use a <script> tag: SuperScript will remove any wrapping tags found inside the block and use only the contents. However, writing a <script> tag invokes intellisense when writing JavaScript.
